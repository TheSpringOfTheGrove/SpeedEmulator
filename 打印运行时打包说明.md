# 打印运行时打包说明

## 目标

打印预览需要调用真诚打印运行时。项目现在不再要求目标电脑必须安装在 `D:\真诚财务软件`，而是优先从程序目录内的 `zhencheng-runtime` 加载。

## 目录约定

开发仓库内放置目录：

```text
D:\Projects\SpeedEmulator\VendorRuntime\Zhencheng\
```

构建或发布后的程序目录：

```text
<程序目录>\zhencheng-runtime\
```

`SpeedEmulator.csproj` 已配置：

```text
VendorRuntime\Zhencheng\**\* -> zhencheng-runtime\**\*
```

所以只要把授权可分发的真诚运行时文件放进 `VendorRuntime\Zhencheng`，`dotnet build` 和 `dotnet publish` 都会自动带到输出目录。

## 加载顺序

程序查找真诚运行时的顺序：

1. 环境变量 `SPEEDEMULATOR_ZHENCHENG_DIR`
2. 程序输出目录 `zhencheng-runtime`
3. 程序输出目录下的 `VendorRuntime\Zhencheng`
4. 当前工作目录 `zhencheng-runtime`
5. 当前工作目录 `VendorRuntime\Zhencheng`
6. 开发机兼容路径 `D:\真诚财务软件`

其中 6 只是开发兼容兜底，正式打包应使用 `VendorRuntime\Zhencheng`。

## 必要文件

以能在真诚程序目录独立运行打印为准，通常需要包含：

- `真诚财务软件.dll`
- `QuestPDF.dll`
- `QuestPdfSkia.dll`
- `SkiaSharp.dll`
- `SkiaSharp.HarfBuzz.dll`
- `HarfBuzzSharp.dll`
- `Stimulsoft*.dll`
- `PdfSharpCore.dll`
- `static\**\*`
- `fonts\**\*`
- 其它同目录的依赖 DLL、native DLL 和模板资源文件

如果缺少依赖，打印模板列表、生成预览或 PDF 导出可能会出现“模板未找到”或运行时加载异常。

## 复制运行时

确认拥有分发授权后，可用脚本从本机真诚安装目录复制运行时：

```powershell
powershell -ExecutionPolicy Bypass -File D:\Projects\SpeedEmulator\tools\Copy-ZhenchengRuntime.ps1
```

自定义来源目录：

```powershell
powershell -ExecutionPolicy Bypass -File D:\Projects\SpeedEmulator\tools\Copy-ZhenchengRuntime.ps1 -Source "D:\真诚财务软件" -Destination "D:\Projects\SpeedEmulator\VendorRuntime\Zhencheng"
```

复制完成后，`VendorRuntime\Zhencheng` 里的文件就是打包源。若需要换电脑后不依赖原安装目录，需要把这些运行时文件随项目提交到仓库或随构建工件保存。

## 本机验证

```powershell
dotnet build D:\Projects\SpeedEmulator\SpeedEmulator.csproj
```

构建后检查：

```powershell
Test-Path D:\Projects\SpeedEmulator\bin\Debug\net8.0-windows\zhencheng-runtime\真诚财务软件.dll
```

返回 `True` 代表运行时已被带到程序输出目录。
