# 极速财务热更新发布说明

## 目标

- 已安装客户端：启动时静默检查更新，有新版本时下载并自动重启到最新版。
- 未安装电脑：通过服务器下载页下载安装包，安装后即可使用，后续走自动更新。
- 发布内容：程序、`Data` 种子数据、模拟用户数据、真诚相关运行文件都会随安装包一起发布。

## 客户端配置

配置文件：`appsettings.json`

```json
{
  "BackendApi": {
    "BaseAddress": "http://159.75.125.68:8088",
    "TimeoutSeconds": 8
  },
  "Update": {
    "Enabled": true,
    "FeedUrl": "http://159.75.125.68/speedemulator/updates/",
    "TimeoutSeconds": 30
  }
}
```

可用环境变量临时覆盖：

- `SPEEDEMULATOR_API_BASE_URL`：后台接口地址。
- `SPEEDEMULATOR_UPDATE_FEED_URL`：热更新源地址。

## 发布新版本

在项目根目录执行：

```powershell
.\tools\Build-HotUpdateRelease.ps1 -Version 1.0.1
```

生成目录：

```text
D:\Projects\SpeedEmulator\artifacts\hot-update\releases
```

这个目录会包含安装包、更新包和 Velopack 更新索引文件。

## 上传到服务器

确认 `C:\Users\admin\.ssh\config` 里有 `Host financial` 后执行：

```powershell
.\tools\Deploy-HotUpdateRelease.ps1
```

默认上传到服务器：

```text
/var/www/html/speedemulator
```

默认访问地址：

- 更新源：`http://159.75.125.68/speedemulator/updates/`
- 安装包：`http://159.75.125.68/speedemulator/download/SpeedEmulator-Setup.exe`
- 下载页：`http://159.75.125.68/speedemulator/download/`

如果服务器 Web 根目录不是 `/var/www/html`，发布时指定目录：

```powershell
.\tools\Deploy-HotUpdateRelease.ps1 -RemoteBase "/你的/web/目录/speedemulator"
```

## 新电脑安装说明

1. 打开下载页：`http://159.75.125.68/speedemulator/download/`
2. 下载 `SpeedEmulator-Setup.exe`。
3. 双击安装。
4. 启动极速财务并登录。

## 注意事项

- 每次发布必须使用递增版本号，例如 `1.0.1`、`1.0.2`。
- 已安装客户端只有在 Velopack 安装环境中才会自动更新；直接运行 `dotnet run` 或发布目录 exe 不会触发安装态更新。
- 更新失败不会阻止登录，会写入本机日志：

```text
%LOCALAPPDATA%\SpeedEmulator\logs\update.log
```
