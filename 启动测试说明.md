# 极速财务前台启动测试说明

本文档说明 `D:\Projects\SpeedEmulator` 这个 .NET 8 WPF 前台项目在开发、构建、发布时的常用操作。

## 1. 启动后台接口

前台登录需要访问后台接口，先确认后台 Docker 服务已经启动：

```powershell
cd D:\Projects\SpeedEmulatorAdmin
docker compose up -d
```

后台地址：

```text
http://localhost:8088
```

## 2. 开发模式启动前台

开发测试时推荐使用 `dotnet run`：

```powershell
cd D:\Projects\SpeedEmulator
dotnet run --project .\SpeedEmulator.csproj
```

这是最常用的本地开发启动方式，会编译并直接启动 WPF 桌面程序。

## 3. 只检查能否编译

`dotnet build` 只负责编译，不是正式打包：

```powershell
dotnet build D:\Projects\SpeedEmulator\SpeedEmulator.csproj
```

默认输出目录：

```text
D:\Projects\SpeedEmulator\bin\Debug\net8.0-windows\
```

编译成功后，也可以直接运行输出目录里的程序：

```powershell
D:\Projects\SpeedEmulator\bin\Debug\net8.0-windows\SpeedEmulator.exe
```

注意：如果 `SpeedEmulator.exe` 已经在运行，再执行 `dotnet build` 可能会失败，因为 exe 文件被当前进程锁定。需要先关闭正在运行的前台程序。

## 4. 正式发布输出

如果需要生成可分发的 Release 目录，使用 `dotnet publish`。

依赖目标电脑已安装 .NET 8 Desktop Runtime：

```powershell
dotnet publish D:\Projects\SpeedEmulator\SpeedEmulator.csproj -c Release -r win-x64 --self-contained false -o D:\Projects\SpeedEmulator\publish
```

如果希望发布目录自带运行时，不要求目标电脑单独安装 .NET 运行时：

```powershell
dotnet publish D:\Projects\SpeedEmulator\SpeedEmulator.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -o D:\Projects\SpeedEmulator\publish
```

发布后的入口：

```text
D:\Projects\SpeedEmulator\publish\SpeedEmulator.exe
```

## 5. 命令区别

```text
dotnet run      开发测试启动程序
dotnet build    只检查项目是否能编译
dotnet publish  正式生成可分发运行目录
```

## 6. 当前项目输出内容

项目文件 `SpeedEmulator.csproj` 已配置以下内容复制到输出目录：

```xml
<Content Include="appsettings.json" CopyToOutputDirectory="PreserveNewest" />
<Content Include="Data\*.json" CopyToOutputDirectory="PreserveNewest" />
```

因此前台运行时需要的配置文件和本地种子数据，会随 `build` 或 `publish` 输出到程序目录。
