# 打印模板印章专项测试报告

生成日期：2026-07-02

## 测试目标

验证所有银行打印模板是否可以正常生成预览，并重点检查用户信息中的印章配置是否能够传入打印模板。

重点场景：

- 用户列表设置的印章图片、印章编码、印章支行需要进入打印模板数据源。
- 农行个人纸质版等依赖用户印章信息的模板，需要能拿到用户配置的印章数据。
- 所有银行模板至少需要通过自动化预览生成，不出现模板缺失、渲染异常或桥接异常。

## 本次修复

修复文件：

- `D:\Projects\SpeedEmulator\Services\ZhenchengPrintBridgeService.cs`

修复内容：

- 将前台用户字段统一桥接到模板运行时字段：
  - `ShouldPrintSeal` -> `IsPrintStamp`
  - `SealImagePath` -> `ZhangImg`
  - `ChapterCode` -> `StampCode`
  - `ChapterBranch` -> `StampBranch`
  - `ChapterBranch` -> `PrintBranch` 兜底
- 同步修复了三条打印路径中的用户数据构造逻辑，避免不同渲染分支字段不一致。

本次专项测试中，当前模板集合没有发现预览生成失败。

用户印章配置已从前台用户信息桥接到模板运行时字段，农行个人纸质版这类依赖用户印章信息的模板现在可以拿到：

## 注意事项

自动化审计验证的是“模板能否正常生成 PDF 预览”，不能完全替代逐个模板的视觉像素级对比。

本次已针对农行个人纸质版相关的印章路径、印章编码、印章支行、印章坐标做了专项探针验证；如果后续发现某个具体模板仍未显示印章，需要优先检查该模板实际读取的字段名是否不同于当前桥接字段。
